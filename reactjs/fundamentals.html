01

What is React.JS?

A JavaScript library for building user interfaces using Components that manage their own State.

Components let you split the UI into independent, reusable pieces, and think about each piece in isolation.

It uses the Virtual DOM and JSX technologies to accomplish that.

React was developed by engineers at Facebook.

into Components

Why React.JS?

It's mostly targeted at projects which need performant DOM manipulations, like Facebook, AirBnB, etc. where a lot of areas update constantly.

It is also a way to work by splitting the interface and app features into components.

What the heck is VirtualDOM?

VirtualDOM
in a nutshell

Whenever something needs to be re-rendered, a virtual representation of the updated DOM is generated.

The Virtual DOM consists of light representations of elements modeled after the component tree, making the process of generating them much more efficient than generating real DOM elements.

Before applying the changes to the real DOM, checks are done to determine where exactly in the component tree the changes happened, a diff is created, and only those specific changes are applied.

  • LIFE CYCLE: Different forms in which a component can exist: mounted, destroyed, etc.

ES6 Modules

Named Exports: Many per module

You cannot assign to imported variables, no matter how you declare the corresponding variables in the module.

lib.js:

                                    
                                    function myApp() {
                                        console.log("Bingo!");
                                    }
                                    const message = "Message!";
                                    export { myApp, message };
                                    export let moduleKey = 12345;

main.js:

                                
                                import { moduleKey, myApp, message } from "./lib.js";
                                myApp();
                                console.log(message);
                                console.log(moduleKey);
                                
                                
                                    => "Bingo!"
                                    => "Message!"
                                    => 12345

JSX: JavaScript XML

Smart Syntax using HTML + JS: JSX elements are treated as JavaScript expressions. 

                                
                                    let jsx = <h1>Title</h1>;

JSX elements can be saved in variables, passed to a function, stored in an object or array...you name it.

                                
                                    let elements = [ <h1>Title</h1>, <p>Text</p> ];

If a JSX expression takes up more than one line, you must wrap it in parentheses.

                                
                                    let header = (
                                        <h1>Title</h1>
                                        <h2>Subtitle</h2>
                                    );

04

Components

Reusable Blocks

A component is a small, reusable chunk of code that is responsible for one job. That job is often to render some HTML.

                                
                                    <App>
                                        <Player></Player>
                                        <Player></Player>
                                    </App>

Every component must come from a component class, which like a factory can produce as many components as you want.

                                
                                    class App extends React.Component { ... }

To make a component class, you use a base class from the React library: React.Component

Gotchas

You cannot inject an if statement into a JSX expressions. Use the logical operators instead: &&, || or ternary operators: ( condition ) ? then : else ;

Avoid Changing the state directly via this.state. Use this.setState() instead.

Don't forget to export the Component.
